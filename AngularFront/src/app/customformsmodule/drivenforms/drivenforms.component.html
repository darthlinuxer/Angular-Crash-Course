Driven Forms

Driven Forms, also known as Template-driven Forms, is an approach for implementing forms in HTML. In Driven Forms, the form's state is primarily managed within the HTML template using directives and two-way data binding.

How Driven Forms work:

  1. Form Element: Start by enclosing your form controls within the <form> element. This element serves as a container for your form.
  2. Form Controls: Inside the <form> element, add various form controls such as text inputs, checkboxes, radio buttons, select boxes, and more. Each form control should have a unique identifier using the id attribute.
  3. Data Binding: Use the [(ngModel)] directive to bind the value of a form control to a property in your component. This enables two-way data binding, meaning changes in the form control update the component property, and changes to the component property update the form control.
  4. Validation: Apply basic validation to form controls using HTML attributes. For example, use the required attribute to make a field mandatory, minlength and maxlength attributes to set length constraints, and pattern attribute to enforce a specific pattern. These attributes can be added to the corresponding form control.
  5. Form Submission: To handle form submission, use the (ngSubmit) event on the <form> element. This event should be bound to a method in your component that handles the form submission logic.
  6. Form Validation and State: Driven Forms automatically track the state of form controls, such as whether they are touched, dirty, or valid. You can use this information to display validation messages or control the form's behavior.
  7. Form Submission Button: Add a submit button inside the <form> element, which users can click to submit the form. You can conditionally disable the button based on the form's validity using the [disabled] attribute.

Driven Forms provide a straightforward way to implement forms in HTML using Angular's built-in directives and two-way data binding. While they are suitable for simple forms with basic validation requirements, for more complex forms, Reactive Forms offer greater flexibility and control over form validation and behavior.
